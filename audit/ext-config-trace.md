## 2. Extension side (manifest.json content_scripts order: constants → cart-signals → base → dom-utils → store-detect → coupon-apply → coupon-fetch → coupon-runner → UI-helpers → inject)

- **`coupon-constants.generated.js`** — build output of `apps/caramel-app/scripts/generate-coupon-constants.ts`, mirrors `lib/coupons.ts` exactly (`window.CaramelCoupons.{STATUSES,VISIBLE_STATUSES,RESTRICTED_STATUSES,STATUS_META}`) — kills the historical status-vocab drift (F-006).
- **`store-detect.js`**: `getDomainRecord()` (`:22-88`) 3-tier fallback — (1) `chrome.storage.local[caramel_supported_stores]` if `Date.now()-ts < TTL` (prod TTL=1h, dev TTL=0, `:14-16`); (2) `runtime.sendMessage({action:'fetchSupportedStores'})` → background fetch, writes cache on success; (3) **expired** cache as last resort (`:70-83`) — only if steps 1-2 both fail does it return `undefined`. `_hostMatchesDomain()` (`:99-109`) requires a label boundary (`.` or `-`) before the domain suffix — blocks `art.com`⊂`walmart.com`-style false matches. `isCheckout()` (`:112-136`) requires **visible**, not merely present, `couponInput`/`showInput`, with one `waitForElement` 3s grace. `tryInitialize()` (`:161-170`) only prompts if `getCachedCodes()` is non-empty — no empty intercept. `startCheckoutDetection()` (`:172-260`): reload-handoff via `sessionStorage.caramel_applied` (discount-link flow), then a `MutationObserver({childList:true,subtree:true})` (`:253-258`) that re-checks visibility on every DOM mutation, debounced 400ms, **self-disconnects after first fire** — this is the sole SPA/drawer-cart safety net (see §5 gap).
- **`inject.js`** (`:1-8`) — fires `startCheckoutDetection()` once on `load`/`readyState==='complete'`. No `history.pushState`/`popstate` listener; SPA route changes rely entirely on the MutationObserver above.
- **`coupon-runner.js`** `startApplyingCoupons()` (`:12-360`): no-rec (`:14-20`), coupon-fetch-throw (`:26-40`), zero-coupons (`:41-49`) all bail to `showFinalModal` with an honest message — none crash. Discount-link capability branch first (`:78-137`, gated on `probeCartJson()`, gets first-class try if the platform exposes `/cart.js`+`/discount/{code}`). DOM-form path: reveals box via `showInput` (2 bounded retries, `:148-166`), bails honestly ("couldn't find the promo box") if still not visible (`:167-180`) instead of grinding 8 dead attempts. Loop caps: `MAX_ATTEMPTS=8` (`:183`), `EARLY_PROBE=2` no-signal early exit (`:198,300-307`), `FLOW_BUDGET_MS=35000` wall-clock backstop (`:209,214-219`). Listener guards `window.__caramel_listeners_bound` (`:365-366`), origin-checked `postMessage` via `CARAMEL_ALLOWED_ORIGINS` (`:369`).
- **`coupon-apply.js`** `applyCoupon()` (`:161-405`): dismiss (`:167`, skipped if `rec.dismissButton` falsy) → reveal input if hidden (`:186-205`, skipped if `rec.showInput` falsy) → fill via native setter + input/change events (`:25-32`) → canonical pointerdown→mousedown→pointerup→mouseup→click (`:261-269`, comment cites two real stores needing this exact order) → `Promise.race([cartSignal(10s), priceTextChange(10s).catch(→'price-nochange')])` (`:319-334`) → success priority: price-drop → committed+stuck → committed+no-errorMsg (`:363-372`). `findAppliedSelector`/`findRemoveSelector` (`:17-22`) fall back to `GENERIC_APPLIED_SELECTORS`/`GENERIC_REMOVE_SELECTORS` (`:6-13`, Honey-style class-substring heuristics) when config omits `successIndicator`/`couponRemove` — **but route.ts already excludes rows missing those**, so this generic path is effectively unreachable via the DB today (dead-but-harmless defense-in-depth). `detectCouponError()` (`:102-154`) diffs against a pre-apply baseline so permanently-mounted aria-live regions don't false-positive forever. `probeCartJson()`/`applyViaDiscountLink()` (`:416-480`) — **runtime capability detection, zero store lists**.
- **`coupon-fetch.js`**: `fetchCoupons()` (`:8-47`) relays through background (avoids CORS). `RESTRICTED_STATUSES` sourced from generated constants (`:54`). `classifyCartCategory()` (`:56-85`) wraps everything in try/catch → `null` on ANY failure (network, non-200, missing `cs.collectCartSignals`). `getCoupons()` (`:90-136`) only calls classify when `hasRestricted` is true (`:109-117`); on classify failure just returns the unannotated list — **coupons are never blocked or hidden by a classify failure**.
- **`UI-helpers.js`**: `insertCaramelPrompt()` (`:8-78`) — single-instance guard `document.getElementById('caramel-small-prompt')` (`:9`), keyboard-parity Enter/Space. `showFinalModal()` (`:201-388`) — 5 terminal copy states (`savedMoney/appliedCode/hasManual/isSignIn/default`, `:254-275`); **`isSignIn=true` is never passed anywhere in the codebase** (grep-confirmed) — that branch and its "Sign in to unlock member-only coupons" copy is dead code today.
- **`dom-utils.js`**: `qOne`/`qAll` (`:176-216`) dual-dispatch CSS vs XPath (`_isXPath`, leading `/`/`(/`/`./`), **wrapped in try/catch → `null`/`[]`** — a malformed/stale selector never throws, it just matches nothing. `pickBestMatch()` (`:52-82`) disambiguates generic selectors by DOM proximity to an anchor, then visibility, then first — no store logic.
- **`background.js`**: `FETCH_TIMEOUT_MS=8000` via `AbortController` (`:26-32`) wraps every network call. `fetchSupportedStores`/`fetchCoupons`/`classifyCart` handlers (`:93-151`) all `.catch()` to a typed empty/error response — **service worker never throws uncaught**. `getActiveTabDomainRecord` (`:152-200`) **always returns `domainRecord:null`** — it only ever resolves the hostname; the popup's coupon fetch is 100% decoupled from the xpath-config path (see §3/§4).
- **`popup.js`**: `initPopup()` (`:48-91`) wrapped in try/catch → `renderLoadError()` (`:95-121`, retry button re-runs `initPopup`) — never blank. **Loader dismissal is a fixed 400ms `setTimeout` (`:38-43`) independent of the actual fetch**, which can take up to 8s — real gap, see §5. `renderUnsupportedSite()` (`:138-198`) is shown identically for "store not tracked" and "supported store, zero coupons right now" — same copy, no distinction (fine, not a hack).

## 3. Three sources of store truth

| Source                                                                     | Contents                                                                                                                                                                                                                                            | Who reads it                                                                                                                                                                                                                    | Freshness                                                                                                   |
| -------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| DB via `/api/extension/supported-stores`                                   | Full xpath contract, `is_active`+`extension_compatible`-filtered                                                                                                                                                                                    | `background.js` → `chrome.storage.local` cache (store-detect.js) — **the only one that actually drives runtime behavior**                                                                                                       | 1h prod / live dev                                                                                          |
| `supported.json` (repo root, `apps/caramel-extension/supported.json:1-24`) | 3 hardcoded domains (amazon/ebay/codecademy)                                                                                                                                                                                                        | **Nothing in the runtime extension fetches it** — repo-wide grep confirms zero references outside `web_accessible_resources` (manifest declaration), `e2e/extension-smoke.spec.ts`, and the old `apple-extension` Xcode project | Frozen since whenever it was last hand-edited — a fossil, not a live source                                 |
| `manifest.json` content_scripts `matches`                                  | Chrome/MV3: `["https://*/*"]` (`:37`) — wildcard, **injects on every https site**, gating is 100% runtime (`getDomainRecord`) not manifest-level                                                                                                    | Browser injection engine                                                                                                                                                                                                        | Static per release                                                                                          |
| `manifest-firefox.json` content_scripts `matches`                          | **STALE**: `["https://www.amazon.com/*","https://*.ebay.com/*","https://*.codecademy.com/*"]` (`:44-48`) — only 3 domains, version `1.0.5` vs Chrome's `1.1.0`, and its `js` array **omits `cart-signals.js`** (`:49-58` vs manifest.json `:39-50`) | Firefox injection engine                                                                                                                                                                                                        | Stale — this is exactly **NF-03** (known finding: manifest/supported-sites drift, the standing red CI test) |

**Disagreement behavior:** Chrome users get correct, DB-driven, universe-wide gating (manifest wildcard + runtime filter). Firefox users are capped at 3 domains **no matter what the DB says** — a DB-added 4th store is simply invisible on Firefox until someone hand-edits `manifest-firefox.json`, and `classifyCartCategory()` silently degrades there too (`window.CaramelCartSignals` undefined → `!cs` guard in `coupon-fetch.js:58` → `null`, graceful but silently weaker). `supported.json` disagreeing with either is inert — it affects nothing a real user sees, only CI (NF-03) and the unmaintained Safari build.

## 5. Robustness gaps (ranked, code-cited) — no per-store hacks found

Repo-wide grep for known store names (amazon/ebay/walmart/logos.com/gapfactory/naturepedic/paragonsports/erincondren/olaplex/etc.) across all 9 core `.js` files returns **zero conditional branches** — every hit is a comment justifying a generic heuristic (pointer-event order, aria-live baselining, visibility-aware reveal). The "no hacks" bar is clean.

1. **Zero production telemetry.** `log()` is a hard no-op in packed builds (`caramel-base.js:78-84`); `recordTiming()` writes only to local `chrome.storage.local['caramel_timings']`, which **nothing ever reads back or transmits** (grep-confirmed, no fetch of that key anywhere). Every silent failure in the case matrix above (stale selectors, API down, no-coupon-box) is invisible to Caramel — only a user complaint would surface it.
2. **iframe-embedded checkout/coupon UI is entirely unreached** — no `all_frames:true` anywhere (`manifest.json:35-52`, `manifest-firefox.json:42-60`). Any hosted-checkout iframe (Bolt/Stripe-style, or an iframed cart drawer) gets no content script at all.
3. **Firefox manifest drift (NF-03 root cause, live today)** — `manifest-firefox.json` hard-caps content-script injection to 3 domains and ships without `cart-signals.js`; a DB-added store is invisible on Firefox regardless of backend state.
4. **Popup loading state is a fixed timer, not a real async gate** (`popup.js:38-43`) — the spinner hides at 400ms flat while `initPopup()`'s network fetch can run up to 8s, leaving a blank `auth-container` window on slow/degraded connections.
5. **MutationObserver only watches `childList`** (`store-detect.js:253-258`) — a pre-rendered, attribute/class-toggled (not DOM-inserted) coupon box on SPA route change won't re-trigger detection.
6. **classify-cart is dead in prod today** (NF-02, missing `OPENROUTER_API_KEY`) — currently exercising the "classify fails" path on every restricted-coupon store in production; confirms the client-side graceful-degrade (§4) is not theoretical.
